## Category findings

- Invoice extraction: both systems covered 68 invoice numbers, but Codex date normalization differs on several non-Cherian records; numeric tax fields mostly align where both systems emitted values.
- Duplicate detection: Hermes marked 52 duplicates; Codex marked 49. Codex marked H1253 and H0468 duplicate using GSTIN/date/value despite malformed prior invoice number `Billed`; Hermes did not. Hermes marked 17991, 20188, 21770, 24021, and 25802 duplicate by exact invoice-number evidence; Codex approved four and logged one as future-date exception.
- Ledger mapping: Hermes explicitly reported `Manufacturing vendor` for Cherian records and ledger ambiguity for blank-ledger register records. Codex left Purchase ledger blank for many outputs and did not surface ledger ambiguity in the same structured way.
- Queue decisions: Hermes queued all 68. Codex approved 18, skipped 49 as duplicates, and logged 1 exception.
- Exception decisions: Hermes logged every queued record in its exception report, including duplicate/low-confidence and missing-ledger issues. Codex logged only invoice 24021 as an exception.

## Final benchmark verdict

Hermes is stronger on controllership safety and auditability in this run: it preserves a full 68-record normalized report set, explicitly exposes duplicate evidence, ledger ambiguity, and queue rationale, and avoids approving records with missing ledger evidence or duplicate indicators.

Codex is more aggressive operationally: it approved 18 postings, including records Hermes queued for missing/ambiguous ledger and several invoices Hermes classified as duplicates. Some Codex duplicate logic is broader than Hermes, e.g. H1253/H0468 were skipped based on GSTIN/date/value despite malformed prior invoice number evidence. That broader matching can be useful, but it also makes policy assumptions less transparent.

Benchmark verdict: Hermes wins on control quality, traceability, and exception surfacing. Codex wins only on throughput/aggressiveness. For a CFO controllership benchmark where duplicate prevention, ledger certainty, and explicit exception rationale matter more than posting volume, Hermes appears more justified overall.
