# Benchmark Data Log

This folder stores each benchmark-question dataset version as a separate CSV file.
This file is the running changelog for how the dataset changes across iterations.

## Update Process

For each new dataset iteration:

1. Add the new file as `data/v<N>.<extension>`, where `N` is the next available version number.
2. Keep prior versions in git history, or retain them as files if side-by-side comparison is needed.
3. Update this file with total questions and distributions for:
   - `service_line`
   - `bi_band`
   - `complexity`
   - `gradability`
4. Include both count and percentage for every represented value.
